# 模型、回测与风险控制

## 1. 评价目标

系统分三层评价，不能相互替代：

1. 概率层：预测概率是否准确、校准。
2. 策略层：按当时赔率筛选后是否具备正期望迹象。
3. 风险层：收益是否由少数偶然事件主导，回撤是否可承受。

命中率只能作为辅助指标。高命中率策略仍可能因赔率过低而亏损。

## 2. 基线与候选模型

### 2.1 必须实现的基线

- 艇位历史频率模型。
- 第一人气/市场隐含概率模型。
- 简单 Logistic Regression 或等价线性概率基线。
- 固定规则策略：只选择低赔率热门，用于揭示“高命中低回报”。

当前第一版无依赖频率基线位于 `boatrace_cal.models.trifecta_frequency`：

- 只使用 `available_at <= prediction_as_of` 的赛果记录训练，避免时间穿越。
- 对 120 个三连单有序组合做拉普拉斯平滑，确保概率和为 1。
- `boatrace-cal frequency-model-candidates` 可为指定比赛输出策略候选 CSV，供 `value-strategy-recommendations` 继续生成回测推荐。
- `boatrace-cal attach-odds-to-candidates` 可把赛前赔率快照按每个候选的 `as_of` 时点写回候选 CSV；没有合法时点赔率时保持空赔率，后续策略会输出 `PASS` 和 `odds_unavailable`。

### 2.2 候选模型

- 梯度提升树类表格模型。
- 排序或成对比较模型。
- 概率融合模型。

模型选择以最终时间外表现、校准和稳定性为准，不以训练集准确率为准。

## 3. 概率校准

模型原始分数必须在独立验证窗口校准。可比较 Platt、Isotonic 等方法，但具体实现取决于获批依赖和样本量。

报告至少包含：

- Log Loss；
- Brier Score；
- Expected Calibration Error；
- 分桶预测概率与真实频率；
- Top-K 覆盖率和命中率；
- 不同场地、月份和赔率区间的校准偏差。

## 4. 时间滚动回测

推荐 Walk-forward 方案：

```text
训练窗口 A -> 验证窗口 A -> 测试窗口 A
                  时间向前滚动
训练窗口 B -> 验证窗口 B -> 测试窗口 B
```

每个测试窗口开始前冻结：

- 数据版本；
- 特征版本；
- 模型及校准器；
- 价值阈值；
- 投注单位和风险上限；
- 赔率选择规则；
- 取消及异常比赛处理规则。

测试窗口结束后才允许更新模型，不得边看结果边修改当期策略。

## 5. 回测执行规则

- 使用预测截点之前最近且未过期的赔率快照。
- 回测前可先执行 `boatrace-cal odds-quality-report`，确认赔率组合覆盖率、快照时效和时间穿越风险。
- 无合法赔率、数据质量失败或无正期望候选时必须弃权。
- 首版使用固定单位投注，降低资金管理对模型效果的干扰。
- 每个组合记录“是否可选、为何选择、当时赔率、结算赔付”。
- 取消、返还、异常和并列情况必须由版本化规则处理。
- 交易成本、赔率滑落或不可成交影响若无法获取，应作为保守敏感性分析列出。

## 6. 策略规则

首版策略可采用保守过滤器：

```text
SELECT 当且仅当：
1. 数据质量门禁通过；
2. 校准概率达到最低阈值；
3. 保守概率下界 × 赔率 - 1 >= 最低优势阈值；
4. 赔率处于训练覆盖充分的区间；
5. 单场候选数量与风险预算未超限。
否则 PASS。
```

阈值必须在验证集确定，并在最终测试前冻结。

当前第一版无依赖实现位于 `boatrace_cal.strategies.value`：

- `implied_probability`、`expected_value` 和 `conservative_expected_value` 提供可测试的市场价值计算。
- `ValueStrategyConfig` 固化最低概率、最低 EV、保守 EV 折扣、最低保守 EV 和最高赔率上限。
- `build_value_recommendation` 输出现有 `Recommendation` 领域对象；满足门禁时固定 `SELECT` 1 单位，否则输出带拒绝原因的 `PASS`。
- `boatrace-cal value-strategy-recommendations` 可从已接入赔率的候选概率 CSV 生成回测可直接读取的 recommendation CSV。

## 7. 风险控制

### 7.1 第一阶段

- 固定单位模拟投注。
- 每场最多选择有限组合。
- 单日总单位上限。
- 同场相关组合合并计入风险。
- 连续亏损不增加投注额。
- 概率或赔率异常时暂停推荐。

### 7.2 后续资金模型

只有在长期纸面模拟通过后，才评估分数 Kelly 等资金模型；即便使用，也应设置显著折扣和硬上限。禁止马丁格尔与亏损追投。

## 8. 指标体系

| 指标 | 作用 |
|---|---|
| 命中率 | 推荐中获胜的比例 |
| 覆盖率 | 有推荐比赛占全部可分析比赛的比例 |
| 回收率 | 总返还 / 总投注 |
| ROI | 净收益 / 总投注 |
| 最大回撤 | 资金曲线最大峰谷损失 |
| 最长连败 | 评估心理和资金压力 |
| Profit Factor | 盈利总额 / 亏损总额 |
| Sharpe 类指标 | 风险调整表现，仅作辅助 |
| Bootstrap 置信区间 | 衡量结果不确定性 |

必须同时展示投注数量和总样本量，避免小样本百分比误导。

## 9. 防过拟合检查

- 最终测试集在所有策略冻结前不可见。
- 同时报告所有预先定义的切片，不挑选最好场地。
- 检查收益是否主要来自极少数高赔率命中。
- 对价值阈值做邻域敏感性分析；仅单点优秀通常不稳定。
- 对赔率下降、预测误差和遗漏比赛进行压力测试。
- 与无模型策略及市场基线对比。

## 10. 上线门槛

### 离线研究 -> 纸面模拟

- 数据泄漏测试通过。
- 至少一个候选模型在多个滚动窗口优于概率基线。
- 主要指标可复现。
- 回测结算与人工样例一致。

### 纸面模拟 -> 小额试验

- 连续多个自然月没有严重校准漂移。
- 实时数据可用率和赔率时效满足门槛。
- 实时模拟与离线回测差异得到解释。
- 用户明确确认预算、损失上限及合规边界。

### 暂停条件

- 数据质量异常或抓取延迟超限；
- 概率校准显著偏离；
- 实际赔率与回放赔率系统性不一致；
- 最大回撤或连续亏损超过预设阈值；
- 比赛规则、数据格式或市场机制发生变化。
