# UI 审核动作第一版计划

## 背景

`codex/ui-smart-table-redesign` 已经把第一版 UI 从概览看板调整为智能表格工作台，并把推荐快照写入回测报告。下一步需要让表格从“可看”推进到“可操作”，但当前项目还没有真实的审核持久化 API。

## 本轮目标

- 在前端本地状态中支持人工审核动作：确认候选、PASS、备注编辑、模拟单位调整、重置审核。
- 保留回测报告中的 `settlement` 和 `recommendation` 作为不可变事实，不把人工审核状态写回样例回测结果。
- 在界面上清楚区分模型推荐、人工审核状态和当前页面内的临时修改。
- 补充 React 交互测试，覆盖审核动作、备注、单位调整和筛选。

## 非目标

- 不实现后端持久化 API。
- 不修改回测结算、推荐快照或样例报告结构。
- 不新增第三方依赖。
- 不实现真实导出 Excel 或确认明日清单。

## 未来 API 契约草案

后续持久化可新增独立审核资源，而不是覆盖 backtest report：

```json
{
  "recommendation_id": "sample-rec-hit",
  "review_decision": "confirmed",
  "stake_units": 1,
  "notes": "人工复核备注",
  "reviewed_at": "2026-07-11T00:00:00Z",
  "reviewed_by": "local-user"
}
```

允许的 `review_decision` 初始集合：`pending`、`confirmed`、`pass`。

## 验收标准

- 默认进入页面时所有候选仍显示为“待审核”。
- 点击“确认候选”后，当前行显示“已确认”，并能通过“已确认”筛选看到。
- 点击“PASS”后，当前行显示“已PASS”，决策列变为 `PASS`，并能通过 `PASS` 筛选看到。
- 模拟单位可在 0 以上加减，不能变成负数。
- 备注可编辑，并同步显示在表格备注列。
- 重置审核会恢复该行的初始单位、备注和“待审核”状态。
- 风险声明继续显示：历史表现不代表未来结果；本系统只提供分析与回测，不承诺盈利，不提供自动下单。

## 验证

- `npm run test`
- `npm run build`
- 若时间允许，再运行 Python 侧 `pytest`、`ruff`、`mypy` 确认本轮未影响后端。
