# 实施路线图

## 1. 实施策略

优先证明数据可信和回测正确，再优化模型。项目按可独立验收的里程碑推进，每个阶段都能产生可复用产物。

## 2. 里程碑

### M0：项目骨架与契约

目标：建立可测试、可配置、可追溯的基础工程。

工作项：

- 初始化 Python 包、测试、配置和日志目录。
- 定义 `race_id`、票种、组合及时间字段规范。
- 定义项目自有的 `payouts` 输入契约和独立导入适配器。
- 建立数据质量检查和样例夹具。
- 固化模型、策略和数据版本字段。

验收：

- 能从可配置路径导入一份符合契约的 `payouts` 文件并生成规范化记录。
- 重复导入幂等，主键冲突有清晰错误。
- 单元测试覆盖组合规范化和比赛主键解析。

### M1：历史数据仓与审计

目标：形成可用于结算和基础统计的可信历史库。

工作项：

- 批量导入比赛结果和赔付。
- 建立采集元数据、异常响应隔离区、标准化表和数据集清单。
- 输出按日期、场地、票种的完整性报告。
- 建立异常、取消和无法结算比赛清单。

验收：

- 随机抽取比赛可追溯到原始来源。
- 官方结果与本地结算样例一致。
- 数据质量报告可重复生成。

### M2：赛前数据与赔率快照

目标：获得真正可用于预测的因果安全数据。

工作项：

- 接入赛程、选手、机艇、展示和环境数据。
- 在预定截点保存赔率快照。
- 为所有字段记录 `observed_at` 与 `available_at`。
- 建立延迟、缺失和格式变化监控。

验收：

- 一场比赛可回放任一预测截点看到的数据。
- 自动测试能识别截点之后的非法记录。
- 赔率组合覆盖率与快照延迟有日报。

### M3：基线模型与概率报告

目标：建立可信、可比较的概率预测基准。

工作项：

- 构建艇位频率、市场和线性模型基线。
- 实现时间切分与滚动训练。
- 实现概率校准和分桶报告。
- 输出三连单 120 类概率及 Top-K 分析。

验收：

- 每个模型能在固定数据版本上复现指标。
- 概率和接近 1，异常值检查通过。
- 报告包含 Log Loss、Brier、ECE 和基线对比。

### M4：价值策略与回测引擎

目标：验证概率优势能否转化为可执行策略。

工作项：

- 计算隐含概率、EV 和保守 EV。
- 实现 `SELECT/PASS`、固定单位和风险上限。
- 实现赔率时点回放与官方赔付结算。
- 输出资金曲线、回收率、最大回撤和切片报告。

验收：

- 已知样例的推荐与结算结果完全一致。
- 策略阈值只由训练/验证窗口决定。
- 报告显示所有预定义切片及置信区间。

### M5：纸面模拟与操作界面

目标：在实时条件下验证数据、推理和离线假设。

工作项：

- 定时执行赛前分析并保存不可变推荐。
- 提供 FastAPI/OpenAPI 和 React 智能表格查看次日预案、最终清单与单场解释。
- 赛后自动结算纸面推荐。
- 建立数据、校准和收益漂移告警。

验收：

- 推荐记录包含完整版本与数据截点。
- 实时结果可与离线回放逐场对照。
- 无候选时系统稳定输出 `PASS`。

## 3. 建议首个迭代

首个开发迭代只完成 M0，并准备 10 至 50 场人工核验样例。不要在数据契约和结算测试完成前训练正式模型。

建议首批文件：

```text
pyproject.toml
src/boatrace_cal/domain.py
src/boatrace_cal/ingestion/payouts.py
src/boatrace_cal/validation/data_quality.py
tests/test_domain.py
tests/test_payouts_ingestion.py
configs/default.yaml
```

具体配置格式应优先复用已安装能力；未批准新依赖时可以使用标准库 TOML/JSON。

## 4. 待确认的产品参数

这些参数不阻塞项目初始化，但必须在 M2 前冻结：

- 首版票种是否只做三连单。
- 预测截点：开赛前 10 分钟、5 分钟或多个截点。
- 纸面模拟的单场和单日风险单位。
- 最低数据完整率、最低 EV 和最低样本量。
- 历史数据可获得的起始日期。
- 是否需要桌面 GUI，或先以 CLI 与 HTML 报告交付。

## 5. 验证矩阵

| 层级 | 重点 |
|---|---|
| 单元测试 | 主键、组合、时间过滤、EV、结算、风险规则 |
| 集成测试 | 外部数据夹具导入、数据集构建、训练到回测链路 |
| 端到端测试 | 从赛前快照到推荐再到赛后结算 |
| 数据测试 | 唯一性、范围、缺失、时效、泄漏 |
| 模型测试 | 可复现、概率和、校准、基线比较 |
| 观测性 | 抓取延迟、数据覆盖、模型漂移、策略覆盖率 |

## 6. 完成定义

每个里程碑只有在以下条件全部满足后才算完成：

- 需求和数据契约已更新；
- 相关测试及静态检查通过；
- 产物可从干净环境重复生成；
- 失败路径有明确日志或原因代码；
- 已知限制和未验证项写入阶段报告；
- 没有用未来数据验证当前预测。
